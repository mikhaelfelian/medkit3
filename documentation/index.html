MEDKIT3 Documentation

Introduction

MEDKIT3 is a PHP-based HMVC framework designed for medical record systems. It provides a robust foundation for building scalable and maintainable web applications.

Project Structure

medkit3/
├── app/
│   ├── controllers/    # Application controllers
│   ├── models/        # Database models
│   ├── views/         # View templates
│   ├── helpers/       # Custom helper functions
│   └── migrations/    # Database migrations
├── systems/
│   ├── controllers/   # Base controller classes
│   ├── models/        # Base model classes
│   ├── routing/       # Routing system
│   └── helpers/       # Core helper functions
├── public/
│   └── assets/        # Public assets (CSS, JS, images)
├── config/
│   ├── config.php     # Main configuration
│   └── database.php   # Database configuration
└── documentation/     # Project documentation

Installation

  1. Clone the repository
  2. Configure your web server to point to the public directory
  3. Copy config/config.example.php to config/config.php and adjust settings
  4. Import the database schema
  5. Run composer install (if using dependencies)

Configuration

Main configuration settings are stored in config/config.php:

// Database Configuration
define('DB_HOST', 'localhost');
define('DB_NAME', 'db_medkit3');
define('DB_USER', 'root');
define('DB_PASS', '');

// Application Configuration
define('BASE_URL', 'http://localhost/medkit3');
define('ROOT_PATH', dirname(__DIR__));
define('DEBUG_MODE', true);

Routing System

The routing system follows a simple pattern: controller/method/parameters

// Example URLs:
http://localhost/medkit3/pasien           # PasienController->index()
http://localhost/medkit3/pasien/create    # PasienController->create()
http://localhost/medkit3/pasien/edit/1    # PasienController->edit(1)

MVC Architecture

The framework follows the Model-View-Controller pattern:

  • Models: Handle data and business logic
  • Views: Present data to users
  • Controllers: Process user input and coordinate between models and views

Models

Models extend BaseModel and handle database operations:

class PasienModel extends BaseModel {
    protected $table = 'tbl_m_pasien';
    protected $primaryKey = 'id';
    protected $fillable = [
        'kode', 'nik', 'nama', 'nama_pgl', 'no_hp',
        'alamat', 'alamat_domisili', 'rt', 'rw',
        'kelurahan', 'kecamatan', 'kota', 'pekerjaan'
    ];
}

Views

Views use PHP templates with the AdminLTE 3 theme:

// Loading a view from controller:
return $this->view('pasien/index', [
    'title' => 'Data Pasien',
    'data' => $data
]);

Controllers

Controllers extend BaseController and handle request/response:

class PasienController extends BaseController {
    public function __construct() {
        parent::__construct();
        $this->loadModel('Pasien');
    }

    public function index() {
        // Controller logic here
    }
}

Helpers

Available helpers include:

  • ViewHelper: Handles view rendering
  • BaseSecurity: Handles security features
  • Notification: Handles flash messages
  • GenerateNoRM: Generates medical record numbers

Database

The system uses PDO for database operations. Tables include:

  • tbl_m_pasien: Patient records
  • tbl_pengaturan: System settings

Security

Security features include:

  • CSRF Protection
  • Input Sanitization
  • PDO Prepared Statements
  • XSS Prevention
