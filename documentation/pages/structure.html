Project Structure

NUSANTARA HMVC follows a clear and organized directory structure to maintain code modularity and separation of concerns.

Directory Layout

medkit3/
├── app/
│   ├── controllers/     # Application controllers
│   ├── core/           # Core application files
│   ├── models/         # Application models
│   └── views/          # View templates
├── config/             # Configuration files
├── databases/          # Database files
│   └── migrations/     # SQL migration files
├── documentation/      # Framework documentation
├── public/            # Publicly accessible files
│   ├── assets/        # Public assets (CSS, JS, images)
│   └── index.php      # Front controller
├── systems/           # Framework core files
│   ├── controllers/   # Base controller classes
│   ├── models/        # Base model classes
│   ├── routing/       # Routing system
│   └── security/      # Security features
├── migrate.php        # Migration CLI script
└── index.php          # Application entry point

Key Directories

  • app/ - Application-specific code
  • config/ - Configuration files
  • databases/ - Database migrations and seeds
  • documentation/ - Framework documentation
  • public/ - Publicly accessible files
  • systems/ - Core framework files

Module Structure

For larger applications, you can organize code into modules:

app/modules/admin/
├── controllers/
│   ├── DashboardController.php
│   └── UsersController.php
├── models/
│   └── AdminModel.php
└── views/
    ├── dashboard/
    └── users/

app/modules/user/
├── controllers/
├── models/
└── views/

Naming Conventions

  • Controllers: PascalCase with "Controller" suffix (e.g., UserController.php)
  • Models: PascalCase with "Model" suffix (e.g., UserModel.php)
  • Views: lowercase with underscores (e.g., user_profile.php)
  • Config files: lowercase (e.g., database.php)
  • Helper files: lowercase with "_helper" suffix (e.g., form_helper.php)
