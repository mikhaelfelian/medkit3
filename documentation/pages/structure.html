Project Structure

NUSANTARA HMVC follows a clear and organized directory structure to maintain code modularity and separation of concerns.

Directory Layout

project/
├── app/                    # Application code
│   ├── controllers/       # Application controllers
│   ├── models/           # Database models
│   ├── views/            # View templates
│   ├── helpers/          # Custom helper functions
│   └── modules/          # HMVC modules
│
├── systems/               # Framework core
│   ├── core/            # Core classes
│   ├── database/        # Database handlers
│   ├── helpers/         # System helpers
│   ├── libraries/       # Third-party libraries
│   └── security/        # Security components
│
├── public/               # Public directory
│   ├── assets/          # CSS, JS, images
│   ├── uploads/         # User uploads
│   └── index.php        # Entry point
│
├── config/              # Configuration files
│   ├── config.php       # Main configuration
│   ├── database.php     # Database settings
│   └── routes.php       # Custom routes
│
└── documentation/       # Framework documentation

Key Directories

app/

Contains your application-specific code:

  • controllers/ - Application controllers
  • models/ - Database models
  • views/ - View templates
  • helpers/ - Custom helper functions
  • modules/ - HMVC modules for larger applications

systems/

Contains the framework's core files:

  • core/ - Base classes (Controller, Model, View)
  • database/ - Database connection and query builders
  • helpers/ - Common helper functions
  • libraries/ - Third-party libraries
  • security/ - Security components

public/

The only directory that should be publicly accessible:

  • assets/ - Static files (CSS, JavaScript, images)
  • uploads/ - User uploaded files
  • index.php - Application entry point

config/

Configuration files:

// config/config.php
$config = [
    'app_name' => 'Your App Name',
    'base_url' => 'http://localhost/your-app',
    'timezone' => 'Asia/Jakarta',
    'charset' => 'UTF-8',
    'debug_mode' => true
];

// config/database.php
$db_config = [
    'hostname' => 'localhost',
    'username' => 'root',
    'password' => '',
    'database' => 'your_database'
];

Module Structure

For larger applications, you can organize code into modules:

app/modules/admin/
├── controllers/
│   ├── DashboardController.php
│   └── UsersController.php
├── models/
│   └── AdminModel.php
└── views/
    ├── dashboard/
    └── users/

app/modules/user/
├── controllers/
├── models/
└── views/

Naming Conventions

  • Controllers: PascalCase with "Controller" suffix (e.g., UserController.php)
  • Models: PascalCase with "Model" suffix (e.g., UserModel.php)
  • Views: lowercase with underscores (e.g., user_profile.php)
  • Config files: lowercase (e.g., database.php)
  • Helper files: lowercase with "_helper" suffix (e.g., form_helper.php)
